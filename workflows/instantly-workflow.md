# Instantly Workflow

## What Instantly is for in this pipeline

Instantly (instantly.ai) is a cold email sending and deliverability tool — mailbox warmup, sending infrastructure, and sequence automation. Use it purely as the delivery layer for outreach generated by `outreach-email.md` and `followups.md`; it has no role in research, qualification, or writing.

## Data handoff points

**This system → Instantly:**
- Paste the generated first-touch email from `outreach-email.md` into an Instantly campaign as step 1.
- Paste each of the five follow-ups from `followups.md` as subsequent steps, matching Instantly's delay settings to the suggested timing in each follow-up's label (Day 3-4, Day 7-8, etc.).
- Use Instantly's variable/merge-field system ({{firstName}}, {{companyName}}) only for the parts of the message that are genuinely reusable structure — the personalized opening sentence in each email should remain hand-written per prospect based on `lead-research.md`, not templated, or the entire point of this system's personalization discipline is lost.

**Instantly → this system:**
- Reply and engagement data (opens, clicks, replies) from Instantly should feed back into `crm.md` as part of the Interaction Log, and can inform whether to accelerate, pause, or adjust a follow-up sequence.

## Setup

1. Set up mailbox warmup in Instantly well before sending real campaigns (typically 2–4 weeks) — deliverability infrastructure is outside this system's scope but directly affects whether well-written outreach even reaches an inbox.
2. Create one campaign per meaningfully distinct segment (matching a specific Company Context / ICP variant if you maintain more than one), not one undifferentiated campaign for your whole list.
3. Import your qualified ("Pursue Now") leads only — do not load Nurture or Deprioritized leads into an active sending campaign; track those separately per `docs/workflow.md#handling-nurture-leads`.
4. Build each campaign step from this system's outputs directly rather than Instantly's built-in AI writing features, to preserve the research-grounded personalization this system is built around.

## Gotchas

- Do not rely on Instantly's spintax/variation features as a substitute for real per-prospect personalization — spintax varies surface wording, not the substance of research-backed personalization this system produces.
- Volume and sending-limit best practices (emails per mailbox per day, ramp-up schedules) are an Instantly/deliverability concern, not something this repository's prompts manage — follow Instantly's own deliverability guidance alongside this system's content guidance.
- Respect CAN-SPAM/GDPR requirements for opt-out handling and sender identification in every campaign — see `docs/best-practices.md#a-disclaimer-on-compliance`.
