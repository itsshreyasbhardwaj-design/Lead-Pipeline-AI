# Apollo Workflow

## What Apollo is for in this pipeline

Apollo (apollo.io) is a prospecting database and sequencing tool — use it as the data source feeding `prospect-discovery.md` and `lead-research.md`, and optionally as the sending/sequencing layer for outreach generated by `outreach-email.md` and `followups.md`.

## Data handoff points

**Apollo → this system:**
- Export a filtered company/contact list (by industry, headcount, technographic filters matching your ICP) as CSV, or paste a summary of an Apollo search directly into `prospect-discovery.md` as your sourcing pool.
- Pull individual contact/company records (title, company, recent job changes, funding data Apollo surfaces) as raw input to `lead-research.md` — treat Apollo's own data points as a starting point to verify, not as pre-verified fact, per the accuracy rules in `prompts/system-prompt.md`.

**This system → Apollo:**
- Paste the generated outreach email/sequence from `outreach-email.md` and `followups.md` into an Apollo Sequence as the first-touch and follow-up steps.
- Log qualification scores and CRM notes from `qualification.md`/`crm.md` into Apollo's contact/account custom fields or notes for team visibility.

## Setup

1. In Apollo, build a saved search using your ICP filters (industry, headcount, geography, technographic signals if available on your plan).
2. Export the list or review it directly, and feed it into `prospect-discovery.md` to get a ranked, signal-prioritized shortlist rather than working the raw export top-to-bottom.
3. For each prioritized company, pull available Apollo data (funding, headcount, contact titles) into `lead-research.md` as a starting point — supplement with direct verification (company website, recent news) since Apollo's firmographic data can lag reality.
4. Once outreach is drafted, create an Apollo Sequence per lead or per segment, using the generated email as step 1 and the five follow-ups from `followups.md` as steps 2–6, adjusted to Apollo's scheduling intervals.

## Gotchas

- Apollo's "buying intent" and technographic signals are probabilistic, not confirmed — always treat them as Strong Inference at best in `lead-research.md`, not Verified fact.
- Contact email accuracy varies by plan tier and data freshness — verify before sending, especially for high-value target accounts.
- Respect Apollo's own usage policies and applicable anti-spam law when sequencing outreach at volume; this system helps you write better messages, it does not manage compliance for you (see `docs/best-practices.md#a-disclaimer-on-compliance`).
- API access and export limits vary significantly by Apollo plan tier — check current limits before planning a large batch discovery run.
